# Training Data

This folder contains the committed training files used by the prototype. They
support the two LLM-facing parts of the system:

- strict intent/slot extraction;
- optional grounded response generation.

The deterministic rule baseline can run without training or downloading any
model. The training files are included so the optional LoRA experiments are
reproducible.

## Slot Extraction Data

`slot_instruction_examples.jsonl` is the older instruction-style slot dataset.
It is retained for the legacy QLoRA path and for comparison.

Rows use:

```json
{"text": "user message", "intent": "search", "slots": {"food": "italian"}}
```

`slot_instruction_examples_augmented.jsonl` is generated by:

```powershell
python scripts/augment_slot_training_data.py
```

`slot_train_strict.jsonl` and `slot_dev_strict.jsonl` are the current strict
training/dev files for `scripts/train_strict_slot_extractor_lora.py`. They use
the runtime-aligned compact JSON target format:

```json
{"input": "book it for friday at 19:00 for 2 people", "target": "{\"intent\":\"book\",\"slots\":{\"day\":\"friday\",\"people\":2,\"time\":\"19:00\"}}"}
```

Regenerate the strict split with:

```powershell
python scripts/build_slot_training_data.py
```

The builder validates JSON targets and rejects exact normalised overlap with
the evaluation fixtures.

## Response Generation Data

`response_generation_examples.jsonl` contains 800 grounded response examples
generated from processed MultiWOZ restaurant records. Standard and challenge
evaluation splits live in `data/evaluation/`.

Rows use:

```json
{"instruction": "...", "input": "Task: ...\nIntent: ...\nUser: ...\nResponse:", "output": "grounded assistant response"}
```

The shared prompt builder is:

```text
src/restaurant_assistant/response_prompt.py
```

That builder is used by response dataset generation, response LoRA training,
runtime response generation and response-model evaluation. The reference
`output` is never included in the model prompt; it is used only as supervised
target text or evaluation evidence.

Regenerate the current split:

```powershell
python scripts/build_response_training_data.py `
  --train-count 800 `
  --eval-count 160 `
  --challenge-count 100 `
  --seed 6062026
```

Then check leakage:

```powershell
python scripts/check_response_data_leakage.py
```

Current generated counts:

| Split | File | Count |
| --- | --- | ---: |
| Train | `data/training/response_generation_examples.jsonl` | 800 |
| Standard eval | `data/evaluation/response_generation_eval.jsonl` | 160 |
| Challenge | `data/evaluation/response_generation_challenge.jsonl` | 100 |

The response builder prefers restaurants that genuinely match stated food,
area and price constraints. Rows labelled as matching are tested to ensure the
evidence records satisfy the prompt constraints.
